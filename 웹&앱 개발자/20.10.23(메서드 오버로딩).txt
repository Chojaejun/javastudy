1. 선결 조건(and)
  - 같은 클래스에서 생선된 메서드들이고
  - 메서드의 이름이 같아야 함.


======메서드 오버로딩 성립 조건(or)
1.파라미터(=매개변수)의 개수가 다르면 메서드 오버로딩 성립

     public void me1( ) {
}
     public void me1(int a ) {
}

2. 파라미터의 개수가 같은데, 자료형이 다르면 메서드 오버로딩 성립

     public void me1(int a ) {
}
     public void me1(long a ) {
}

3. 파라미터의 개수와 자료형이 같아도 파라미터의 자료형 배치 순서가 다르면 오버로딩 성립

     public void me1(int a, Dog b) {
}
     public void me1(Dog a, int b) {
}

===============메서드 오버로딩을 사용해야 하는 때
메서드들의 전반적인 기능이 같은데 
파라미터 자료형이 다를 때...

=================메서드 오버로딩과 관련 없는 것들
1. 접근제한자
2. 반환형
3. 파라미터의 변수명
4. 메서드의 본문 내용

=========메서드 오버로딩의 장점은 메서드 호출 때 나온다.


======메서드 오버로딩 Vs 메서드 오버라이딩에 대해 설명하세요.
오버라이딩: 상속 관계가 성립하는 두 클래스에서 자식클래스가 부모클래스의 
               메서드 내용을 재정의(수정/삭제/추가)하는 것을 의미함.
